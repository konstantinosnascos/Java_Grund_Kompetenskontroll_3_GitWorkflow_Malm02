### Val av Datastrukturer
I **BookingRepository**-klassen valde vi att använda 
oss av Map för **booking** och 
**timeTable** här för att lättare söka igenom listan 
efter våra objekt. Då Map har en **"nyckel"** kan 
vi låta användaren mata in ett **lättare referens**
(än att t.ex behöva skriva in både för och efternamn) 
som vi då kan använda som en nyckel för att hitta den 
eftertraktade objektet.

I **BookingMenu** använder vi oss av vanliga Listor 
som ArrayLists då de används för mer temporära ändamål. Samt.
i **BookingRipository** använder vi Listor för samma anledning.

### Git
Det var första gången vi aktivt jobbat med git i gruppen.
Vi utsåg en person ansvarig för Git main branchen/merge till
main. Dock eftersom vi var ovana så sammarbetade vi med alla
merges vi gjort. Vi har prövat att göra flera branches och hoppa
till nya branches efter att ha kommit till en viss punkt i 
projektet. Vi valde att fortsätta hålla vissa branches uppdaterade.

### Genomfört Arbete:
Vi har lyckats uppfylla samtliga krav i projektet. Alla kraven från F1 till F18 ska vara fungerande.

### Kvarvarande Arbete:
Vi hann med alla kraven så det finns inget kvar som ligger kvar för oss. Dock så kan man göra koden 
snyggare såklart och det finns lite kod som inte används, men vi lämnar några av dem medvetet då vi 
tror att om man vill expandera på projektet senare(som också är ett krav) så kan man göra det. 
Vi tror att det finns potential i de klasser och metoder, men vi valde att prioritera kraven först 
istället för att implementera snyggare och annan ytterligare funktionalitet.
