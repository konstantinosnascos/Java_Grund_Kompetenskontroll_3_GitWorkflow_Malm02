# Rapport

### Val av Datastrukturer
I **BookingRepository**-klassen valde vi att använda 
oss av Map för **booking** och 
**timeTable** här för att lättare söka igenom listan 
efter våra objekt. Då Map har en **"nyckel"** så kan 
vi låta användaren mata in ett lättare referens som 
vi då kan använda som en nyckel för att hitta den 
eftertraktade objektet.

I **BookingMenu** vi oss av vanliga Listor som ArrayLists
och LinkedLists då de används för mer temporära ändamål. LinkedLists
används för att lägga in flera gånger och kanske senare kunna också
ta bort enligt användarens önskningar.

### Git
Det var första gången vi aktivt jobbat med git i gruppen.
Vi utsåg en person ansvarig för Git main branchen/merge till
main. Dock eftersom vi var ovana så sammarbetade vi med alla
merges vi gjort. Vi har prövat att göra flera branches och hoppa
till nya branches efter att ha kommit till en viss punkt i 
projektet. Vi valde att fortsätta hålla vissa branches uppdaterade.
